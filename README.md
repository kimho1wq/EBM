# EBM
This repository contains the official PyTorch implementation of the EBM framework proposed in the following papers:
- Conference version
  - Title: Enhancing Audio Deepfake Detection by Improving Representation Similarity of Bonafide Speech
  - Venue: Interspeech 2025
  - Authors: Seung-Bin Kim, Hyun-Seo Shin, Jungwoo Heo, Chan-Yeong Lim, Kyo-Won Koo, Jisoo Son, Sanghyun Hong, Souhwan Jung, and Ha-Jin Yu
  - DOI: https://doi.org/10.21437/Interspeech.2025-422
- Journal version
  - Title: Comprehensive Validation of Bridge Module and EBM Loss for One-Class Audio Deepfake Detection
  - Journal: IEEE Access, Vol. 14, pp. 54179–54191, 2026
  - Authors: Seung-Bin Kim, Hyun-Seo Shin, Jungwoo Heo, Chan-Yeong Lim, Kyo-Won Koo, Jisoo Son, Souhwan Jung, and Ha-Jin Yu
  - DOI: https://doi.org/10.1109/ACCESS.2026.3680160

Our experimental code was modified based on [HM-Conformer](https://github.com/talkingnow/HM-Conformer).


# Overview
![Image](https://github.com/user-attachments/assets/63538b35-8835-46bb-9140-4399f5b6cad4)

Audio deepfake detection aims to distinguish bona fide speech from spoofed speech generated by text-to-speech, voice conversion, and other speech synthesis techniques.
A key challenge is that spoofing attacks observed during evaluation can differ significantly from those seen during training, which makes generalization to unseen attacks difficult.

The proposed Enhanced Bona fide Modeling (EBM) framework addresses this issue from a one-class learning perspective.
Instead of relying only on a binary decision boundary between bona fide and known spoof samples, EBM explicitly models the bona fide distribution as a compact and coherent region in the embedding space. At the same time, spoofed samples are pushed away from this bona fide region.

![Image](https://github.com/user-attachments/assets/cc82a14d-c070-4dc4-b4f5-ff8fe4145251)
The EBM objective consists of two main components:

#### 1. Bonafide-Pair Learning (BPL) loss
  - Reduces the intra-class variance of bona fide speech.
  - Aligns embeddings from two augmented views of the same bona fide utterance.
  - Encourages bona fide samples to form a more compact representation space.
#### 2. Extended One-Class Softmax (EOC-S) loss
  - Extends the one-class softmax formulation for audio deepfake detection.
  - Uses a bona fide-centered representation space.
  - Updates the centroid mainly using bona fide samples.
  - Applies margin constraints to separate bona fide and spoofed samples more explicitly.

Together, BPL and EOC-S improve the representation similarity of bona fide speech while enhancing separability from spoofed speech.


# Journal Version: What Was Extended?
![Image](https://github.com/user-attachments/assets/0e473c04-d303-46c1-a843-e86b20f87330)
The Interspeech version introduced the initial EBM framework, focusing on improving the representation similarity of bona fide speech for audio deepfake detection.

The IEEE Access journal version further extends and validates the framework more comprehensively. In particular, the journal version investigates the following aspects:

#### 1. Bridge module validation
  - The bridge module adaptively aggregates multi-layer representations from a frozen self-supervised speech frontend.
  - It uses layer-wise attention and attentive statistics pooling to construct a compact utterance-level representation.
  - The journal version validates the effectiveness of this bridge module across different model configurations.
#### 2. Data augmentation analysis
  - The journal version compares multiple augmentation strategies, including noise-based augmentation and non-noise transformations.
  - Examples include Random Color Noise, RawBoost, MUSAN, speed perturbation, codec transformation, and reverberation.
#### 3. Hyperparameter sensitivity
  - The effects of EBM-related hyperparameters are investigated.
  - This includes the weights and margin settings used in the loss function.
#### 4. Train-test utterance length analysis
  - The journal version analyzes how training and evaluation utterance lengths affect detection performance.
  - This is important because audio deepfake detection systems are often deployed under varying utterance duration conditions.
#### 5. Portability across frontends and backbones
  - The EBM framework is evaluated across different self-supervised frontends and backbone architectures.
  - This verifies whether the proposed method is tied to a specific architecture or can be applied more generally.


# Data
The following datasets were used in our experiments:
  - [ASVspoof 2019 Logical Access](https://www.asvspoof.org/index2019.html)
  - [ASVspoof 2021 Logical Access & DeepFake](https://www.asvspoof.org/index2021.html)

The ASVspoof 2019 LA training set consists of:
  - 2,580 bona fide samples
  - 22,800 spoof samples
The ASVspoof 2021 LA and DF evaluation sets were used to evaluate generalization under more challenging and realistic conditions.

# Vocoder Augmentation

We additionally applied vocoder-based augmentation to the training data using [HiFi-GAN](https://arxiv.org/pdf/2010.05646).
In this repository, HiFi-GAN augmentation is applied only to spoof samples.
The procedure for applying HiFi-GAN follows the official implementation [here](https://github.com/jik876/hifi-gan).


# Environment
Docker image (nvcr.io/nvidia/pytorch:23.08-py3) of Nvidia GPU Cloud was used for conducting our experiments.

Make docker image and activate docker container.
```
.docker_build.sh
.docker_run.sh
```

Note that you need to modify the mapping path (/data) before running the 'docker_run.sh' file.
Additionally, the dataset path is specified in the `arguments.py` file, so it must be set accordingly:

For example:
  - -v /your/local/data/path:/data

In `arguments.py`
```
'path_19LA'  : '/data/ASVspoof2019',
'path_21LA'  : '/data/ASVspoof2021_LA_eval',
'path_21DF'  : '/data/ASVspoof2021_DF'
```

We have a basic logger that stores information in local. However, if you would like to use an additional online neptune logger:
```
# Neptune: Add 'neptune_user' and 'neptune_token'
# input this arguments in "system_args" dictionary:
# for example
'neptune_user'  : 'user-name',
'neptune_token' : 'NEPTUNE_TOKEN'
```
        

# Training
- Trining on a single GPU
```
CUDA_VISIBLE_DEVICES=0 python3 main.py
```
- Trining on multiple GPUs
```
CUDA_VISIBLE_DEVICES=0,1 python3 main.py
```

# Repository Structure

A typical structure of this repository is as follows:

```
EBM/
├── main.py
├── arguments.py
├── dataset/
├── model/
├── loss/
├── utils/
├── docker_build.sh
├── docker_run.sh
└── README.md
```

Main components:
  - main.py: training and evaluation entry point
  - arguments.py: configuration file for paths, hyperparameters, and logging
  - model/: model architecture, frontend, bridge, and backbone modules
  - loss/: EBM-related loss functions, including BPL and EOC-S
  - dataset/: dataset loading and preprocessing
  - utils/: utility functions

The exact directory structure may vary depending on the current version of the repository.

## Implementation Notes and Hyperparameter Clarification

Please note that there is a minor inconsistency in the hyperparameter description of the paper. Although the paper states that $\beta_1=20$ and $\beta_2=5$ were used, the reported EBM performance was actually obtained with $\beta_1=5$ and $\beta_2=20$.

In the follow-up journal version, *Comprehensive Validation of Bridge Module and EBM Loss for One-Class Audio Deepfake Detection*, additional hyperparameter tuning was conducted. Based on this tuning, we found that $batch=40$ and $\beta_1=30$ provided better performance in our experimental setting. Therefore, for reproduction or further experiments, it is also reasonable to use $batch=40$ and $\beta_1=30$.

However, please note that the optimal hyperparameter values may vary depending on the dataset, augmentation strategy, and experimental environment.
